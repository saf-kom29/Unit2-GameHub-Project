ü§î What Are Media Queries?

Media queries let you apply different CSS based on screen size 


/* Default styles for desktop */
.main-nav {
    display: flex;
}

/* Different styles for mobile (768px and smaller) */
@media (max-width: 768px) {
    .main-nav {
        display: none;  /* Hide on mobile */
    }
}
**MEDIA QUERIES** - Conditional CSS based on screen size - @media rule - max-width = "up to this size" - Desktop-first approach

üîç Anatomy of a Media Query


@media (max-width: 768px) {
    /* CSS rules that apply when screen is 768px or smaller */
    .hero {
        padding: 60px 0;
    }
}

Breaking It Down:

  • @media = Start of media query
  • max-width: 768px = Condition (768px and below)
  • { ... } = CSS rules to apply when condition is true
**ANATOMY** - @media keyword - Condition in parentheses - CSS rules in curly braces - Simple conditional logic

Step 4: The CSS "Trick" ‚ú®


/* When hamburger is focused, show the mobile nav next to it */
.hamburger:focus + .mobile-nav {
    display: flex;  /* Show the menu! */
}

How It Works:

  • :focus = When element is clicked/tapped
  • + = Select the next sibling element
  • .mobile-nav = The menu to show
  • Result: Click hamburger ‚Üí menu appears! üéâ
**THE TRICK** - Adjacent sibling selector (+) - :focus pseudo-class - No JavaScript required - Pure CSS solution!
INSTRUCTOR-LED

Step 5: Create the Media Query

Scroll to the bottom of your CSS file and add:


/* ===================================
   LESSON 14: MEDIA QUERIES

   LESSON 14: Mobile Breakpoint (768px and below)

    /* Hide desktop navigation */
    .main-nav {
        display: none;
    }
    
    /* Show hamburger icon */
    .hamburger {
        display: block;
    }
}
**STEP 5: MEDIA QUERY** - Go to bottom of CSS - Add media query section - Hide desktop nav - Show hamburger

üêõ Common Issues & Fixes

Issue #1: Media query not working

Fix: Check for typos in @media, missing colons, or incorrect values

Issue #2: Styles not showing up

Fix: Make sure media queries are at the BOTTOM of your CSS file

Issue #3: Wrong element selected

Fix: Double-check class names match your HTML exactly

**COMMON ISSUES** - Typos are #1 cause - Order matters - Case-sensitive - Check DevTools console
